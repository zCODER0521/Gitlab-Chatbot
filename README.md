# GitLab Handbook Chatbot

A Generative-AI chatbot that answers questions about GitLab's public
[Handbook](https://handbook.gitlab.com/) and [Direction](https://about.gitlab.com/direction/)
pages. Retrieval-augmented generation with citations and metadata-filtered search,
a Streamlit chat UI, and **open-source Llama** running on **Groq**.

## What it does

1. **Discovers** URLs from GitLab's public sitemaps (~4000 pages) and picks the
   top 50 across categories (engineering, hiring, values, direction, etc.).
2. **Scrapes** those pages and converts them to clean markdown, tagging each
   with a `category` label.
3. **Embeds locally** with `BAAI/bge-small-en-v1.5` (384-dim) and **upserts**
   the raw vectors to **Pinecone** with flat metadata for filtering.
4. On each question: query embedded locally → dense vector search →
   **`cross-encoder/ms-marco-MiniLM-L-12-v2`** local rerank → optional category
   filter → Groq Llama → streamed answer with inline `[1]` citations linked to
   the source page and section.

## Quickstart (local)

Prereqs: Python 3.10+, a free [Groq API key](https://console.groq.com/keys),
and a free [Pinecone API key](https://app.pinecone.io).

```bash
# 1. Install
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. Add your keys
cp .env.example .env
# edit .env and paste GROQ_API_KEY + PINECONE_API_KEY

# 3. Build the knowledge base (one-time, ~3 min)
python -m src.discover     # sitemap → top 50 URLs → data/seed_urls.txt
python -m src.scraper      # fetch + clean → data/raw/*.md
python -m src.ingest       # chunk → embed locally → upsert raw vectors

# 4. Run the chat
streamlit run app.py
```

Then open http://localhost:8501.

## Configuration

Edit `src/config.py` to change models, chunk sizes, or top-k bounds.

| Knob | Default | Notes |
|------|---------|-------|
| `LOCAL_EMBED_MODEL` | `BAAI/bge-small-en-v1.5` | local sentence-transformers, 384-dim |
| `LOCAL_RERANK_MODEL` | `cross-encoder/ms-marco-MiniLM-L-12-v2` | ~33M-param cross-encoder, runs on CPU; 10-15× faster than bge-reranker-v2-m3 |
| `GROQ_DEFAULT_MODEL` | `llama-3.3-70b-versatile` | best quality |
| `GROQ_FAST_MODEL` | `llama-3.1-8b-instant` | used for follow-up generation |
| `CHUNK_SIZE_CHARS` | 2000 | ~500 tokens |
| `CHUNK_OVERLAP_CHARS` | 200 | header-aware splitter handles most boundaries |
| `RETRIEVE_FETCH_K` | 12 | over-fetched before rerank |
| `TOP_K_DEFAULT` | 5 | final results to LLM (tunable in sidebar) |
| `CONFIDENCE_*` | floats in [0, 1] | rerank-score thresholds for refuse / low / high bands |
| `CITATION_SUPPORT_THRESHOLD` | 0.1 | sentence↔chunk score below this → "weakly supported" |
| `INGEST_CATEGORIES` | `["values","culture","hiring","engineering"]` | which categories actually get embedded; set to `None` to ingest all |
| `PERSONAS` | dict | sidebar persona presets (style hint + default categories + starter prompts) |

## Updating the knowledge base

Re-run discovery + scrape + ingest. Pinecone upserts are idempotent (records
with the same `_id` are overwritten).

```bash
python -m src.discover && python -m src.scraper && python -m src.ingest
```

To tune what gets ingested, edit `CATEGORY_QUOTAS` and `TOP_N_URLS` in `src/config.py`.

## CLI usage (no UI)

```bash
python -m src.retriever "What are GitLab's values?"          # top-k after rerank
python -m src.rag "What are GitLab's values?"                # full grounded answer
python -m src.rag --cat hiring "What's the interview process?"  # category filter
python -m src.llm                                            # smoke-test Groq
```

## Metadata schema (Pinecone)

Each record in the `gitlab-handbook` index carries:

| Field | Type | Notes |
|---|---|---|
| `_id` | string | `<page-slug>-<chunk-index>` |
| `text` | string | the chunk; auto-embedded by Pinecone |
| `source_url` | string | full URL of the source page |
| `title` | string | page title |
| `section_path` | string | e.g. `Engineering > Hiring > Interview Process` |
| `category` | string | one of: values, culture, hiring, people, engineering, product, direction, commercial, leadership, legal_security, general |
| `chunk_index` | number | position within the source page |

## Deployment (Streamlit Community Cloud)

1. Push this repo to GitHub.
2. At [streamlit.io/cloud](https://streamlit.io/cloud) → **New app** → point at
   `app.py`.
3. Under **Secrets**, add:
   ```toml
   GROQ_API_KEY = "your_key_here"
   PINECONE_API_KEY = "your_key_here"
   ```
4. The Pinecone index is hosted, so the deployed app just queries it — no local
   index to ship. Run `python -m src.discover && src.scraper && src.ingest`
   once locally to seed the index before deploying.

## Bonus features

Beyond the base RAG loop, this build adds:

- **Out-of-corpus refusal with confidence badges** — the top reranker score is
  classified into `refuse / low / medium / high` bands. Below the refuse
  threshold the LLM is never called; otherwise the answer is shown with a
  coloured **Confidence** badge so the user can calibrate trust at a glance.
  Thresholds live in `src/config.py` (`CONFIDENCE_*`).
- **Citation faithfulness check** — after streaming, every `[N]` is parsed
  out and the sentence around it is scored against the cited chunk via the
  same cross-encoder. The result is summarised as a single
  `Grounded ✓ / Partially grounded ⚠ / Unsupported ✗` badge, with a
  per-citation breakdown in verbose mode (`src/guardrails.py`).
- **Follow-up question chips** — after each answer the fast Groq model
  proposes three contextual follow-ups, rendered as clickable buttons
  (`src/rag.py::suggest_followups`).
- **Persona selector** — a sidebar radio (`New hire / Engineer / Hiring
  manager / Aspiring employee / Just exploring`) tunes the assistant's tone,
  pre-fills the category filter, and swaps the empty-state starter prompts
  (`src/config.py::PERSONAS`, `src/prompts.py::system_prompt_for`).
- **UX scaffolding** — model prewarm with a step-by-step status widget,
  scrollable fixed-height chat history, verbose mode that surfaces
  retrieve / rerank / LLM timings and full untruncated source text.
